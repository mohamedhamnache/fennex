# Fennex i18n — Multi-Language UI Support

**Date:** 2026-06-29
**Status:** Approved

## Goal

Translate all dashboard UI strings into multiple languages. The user selects their language via a TopBar picker or Settings > Account; the preference is saved to their profile so it syncs across devices.

---

## 1. Supported Languages (initial)

| Code | Name | Flag |
|------|------|------|
| `en` | English | 🇬🇧 |
| `fr` | Français | 🇫🇷 |
| `es` | Español | 🇪🇸 |
| `de` | Deutsch | 🇩🇪 |
| `pt` | Português | 🇵🇹 |
| `ar` | العربية | 🇸🇦 |

Adding a new language in the future requires only dropping a `public/locales/{code}/common.json` file and adding one entry to the `SUPPORTED_LOCALES` constant.

## 2. Architecture

```
react-i18next (i18next + react-i18next + i18next-browser-languagedetector + i18next-http-backend)

apps/web/
  public/locales/
    en/common.json          ← source-of-truth English strings
    fr/common.json
    es/common.json
    de/common.json
    pt/common.json
    ar/common.json
  lib/
    i18n.ts                  ← i18next init; reads fennex_lang cookie
  components/
    providers/
      I18nProvider.tsx       ← wraps root layout; syncs user.language from profile
    layout/
      LanguagePicker.tsx     ← dropdown in TopBar; also used in Settings
  app/
    (dashboard)/
      settings/page.tsx      ← gains Language row in Account section
```

### Load flow

1. Browser requests page → `i18n.ts` reads `fennex_lang` cookie → inits i18next with that locale (no visible flash).
2. `I18nProvider` mounts → `useQuery("me")` resolves → if `user.language` differs from current locale, call `i18n.changeLanguage(user.language)` + rewrite cookie (cross-device sync).
3. User picks a language (TopBar or Settings) → `i18n.changeLanguage(code)` → write cookie → `PATCH /users/me { language: code }` → all `t()` calls re-render in-place.

### Cookie

- Name: `fennex_lang`
- Set via `document.cookie` (client-side only)
- Max-age: 1 year; `path=/; SameSite=Lax`

## 3. Backend Changes

### 3a. User model — new column

```python
# apps/api/app/models/user.py
language: Mapped[str] = mapped_column(String(5), nullable=False, server_default="en")
```

### 3b. Alembic migration

```sql
ALTER TABLE users ADD COLUMN language VARCHAR(5) NOT NULL DEFAULT 'en';
```

### 3c. Schema updates

`UserProfile` (returned by `GET /users/me`) gains:
```python
language: str
```

New Pydantic request schema:
```python
class UpdateLanguageRequest(BaseModel):
    language: str   # validated against SUPPORTED_LOCALES

SUPPORTED_LOCALES = {"en", "fr", "es", "de", "pt", "ar"}
```

### 3d. New endpoint

```
PATCH /api/v1/users/me/language
Body: { "language": "fr" }
Returns: { "language": "fr" }
```

Validates `language` is in `SUPPORTED_LOCALES`; raises `400` otherwise. Updates `current_user.language` and commits.

## 4. Frontend — `lib/i18n.ts`

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";
import Backend from "i18next-http-backend";

export const SUPPORTED_LOCALES = ["en", "fr", "es", "de", "pt", "ar"] as const;
export type Locale = (typeof SUPPORTED_LOCALES)[number];

i18n
  .use(Backend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: "en",
    supportedLngs: SUPPORTED_LOCALES,
    ns: ["common"],
    defaultNS: "common",
    backend: { loadPath: "/locales/{{lng}}/{{ns}}.json" },
    detection: {
      order: ["cookie", "navigator"],
      lookupCookie: "fennex_lang",
      caches: ["cookie"],
      cookieOptions: { maxAge: 365 * 24 * 3600, path: "/", sameSite: "Lax" },
    },
    interpolation: { escapeValue: false },
  });

export default i18n;
```

## 5. Frontend — `I18nProvider.tsx`

```typescript
"use client";
import { useEffect } from "react";
import { I18nextProvider } from "react-i18next";
import { useQuery } from "@tanstack/react-query";
import i18n from "@/lib/i18n";
import { getMe, isAuthenticated } from "@/lib/api";

export function I18nProvider({ children }: { children: React.ReactNode }) {
  const { data: me } = useQuery({
    queryKey: ["me"],
    queryFn: getMe,
    enabled: isAuthenticated(),
    staleTime: 5 * 60_000,
  });

  useEffect(() => {
    if (me?.language && me.language !== i18n.language) {
      i18n.changeLanguage(me.language);
    }
  }, [me?.language]);

  return <I18nextProvider i18n={i18n}>{children}</I18nextProvider>;
}
```

Mounted in `app/layout.tsx` wrapping all children.

## 10. Non-Goals

- Server-side rendering with locale (all client components; no RSC locale needed)
- RTL layout for Arabic (strings are translated; full RTL CSS mirroring is a follow-up)
- Translating AI-generated content (articles, social posts) — that's controlled by the project's `locale` field
- Machine-translation of the JSON files (translations written manually / by the team)
